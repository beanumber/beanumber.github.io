Data in the Intro Course

  • sampling: simple random sample

  • structure: flat file (e.g. rows and columns)

  • integrity: no missing data

  • size: maybe a few hundred rows, one or two dozens columns

  • timeframe: fixed snapshot

  • type: numerical and categorical

  • Do we teach it this way because:
    • that's the kind of data we generate?
    • that's the kind of data that would be generated by the models we want to fit to it?

Data in real life

  • sampling: observational

  • structure: relational database, XML, scraped from WWW

  • integrity: ?

  • size: thousands to millions of rows, hundreds of columns

  • timeframe: live stream, or regularly updated

  • type: text, images, or video

The Course

Big Picture

  • Neither exclusively statistics, nor exclusively computer science

  • Not just a grab-bag of topics from each

  • Fits into larger movement in stats curriculum:
    • statistical programming (Nolan and Temple Lang, 2010)
    • "data habit of mind" (Finzer, 2013)
    • tidy data, data manipulation "verbs" (Wickham, 2014)
    • statistics is reasoning, not models (Moore, 1998)

SQL

  • Learn basic SQL querying syntax (e.g. SELECT)

SQL vs. R

  • Learn to choose appropriate tool

  • Filter by rows & columns
    • SELECT col1, col2 FROM a WHERE col3 = 'x'
    • subset(a, col3 == "x", select = c(col1, col2))
  • Aggregate by rows
    • SELECT id, sum(col1) as mysum FROM a GROUP BY id
    • summarise(group_by(a, id), mysum = sum(col1))
  • Combine two tables
    • SELECT * FROM a JOIN b ON a.id = b.id
    • merge(x=a, y=b, by="id"))

Practice

Project

  • Semester-long, very open-ended

  • Rules:
    • Your project must be centered around data
    • Your project must tell us something
    • To get an A, you must show something beyond what we've learned in class

  • Deliverable: compilable R Markdown document

Data Expo

  • Before projects start, but after groups formed

  • A full class period, invite faculty & staff from:
    • Engineering
    • Spatial Analysis Lab
    • Administrative Technology
    • Institutional Research
    • National Priorities Project

  • Tell students about the data you work with the challenges you face

  • Hugely motivating for students

Project A: weathR

  • Q: what can Twitter tell us about how the weather is perceived in different places?
    • scrape, store, and analyze Twitter data using API
    • monitor scripts over two-month period
    • create visualizations
    • wrote Shiny web application

  • high technical degree of difficulty

  • creativity and planning

Project B: Smathies

  • Q: what does the network of Smith math major graduates look like?
    • interviewed math faculty for major "database"
    • scraped data from MathSciNet
    • requested and obtained SQL database from Math Genealogy Project
    • used igraph package to construct network
    • wrote Shiny web application

  • high technical degree of difficulty

  • great teamwork and "soft skills"
